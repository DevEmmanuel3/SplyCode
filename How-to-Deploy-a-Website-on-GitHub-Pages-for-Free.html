How to Deploy a Website on GitHub Pages for Free

Want to get your website online for free? GitHub Pages offers a simple way to publish static websites directly from your GitHub repository. Whether you're a beginner or an experienced developer, it's a great option for portfolios, documentation, and personal projects. Here's a step-by-step guide to help you deploy your site.

What is GitHub Pages?

GitHub Pages is a free web hosting service provided by GitHub that allows you to host HTML, CSS, and JavaScript files directly from a GitHub repository. It's ideal for static websites and supports custom domains and HTTPS.

Step 1: Create a GitHub Repository

  1. Go to github.com and sign in.
  2. Click the "+" icon and select "New repository".
  3. Name your repository (e.g., my-website).
  4. Check "Add a README file".
  5. Click "Create repository".

Step 2: Add Your Website Files

You can do this in two ways:

Upload via GitHub Interface:

  • Click "Add file > Upload files".
  • Drag and drop your HTML, CSS, and JavaScript files.

Use Git on Your PC:

git clone https://github.com/yourusername/my-website.git
cd my-website
# Copy your files into this folder
git add .
git commit -m "Add website files"
git push origin main

Step 3: Enable GitHub Pages

  1. Go to your repository.
  2. Click on "Settings" > "Pages" in the sidebar.
  3. Under "Source", choose "main" branch and click "Save".
  4. Your website will be published at https://yourusername.github.io/my-website/.

Step 4 (Optional): Use a Custom Domain

  1. Register a domain name.
  2. In your DNS settings, add a CNAME record pointing to your GitHub Pages URL.
  3. In your repository, create a CNAME file with your custom domain (e.g., www.yoursite.com).

Benefits of GitHub Pages

  • Completely free
  • Built-in HTTPS
  • Custom domain support
  • Fast and reliable
  • Perfect for developers and students
